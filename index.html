How IPTV Detective works — simple explanation

This site helps you check an IPTV provider's channel list and compare it with other public submissions to find identical or very similar packages. It's useful for spotting resellers (different shops selling the same package) and comparing prices.

How to use it
  1. Open the Check Provider tab and enter the provider name, provider website URL, and the yearly price.
  2. Choose the provider's .m3u file (the playlist that lists channels). If the file is large, parsing may take a minute.
  3. Submit the form. The site compares channels and groups in your file to other submissions and shows:
    • Channel & group counts — how many channels and groups were found.
    • Fingerprint (hash) — a short deterministic hash used to find identical files.
    • Similarity/compare result — matching providers and a visual match score.
Privacy — what we send

We do not upload the full M3U file to the server for most checks — only small counts and fingerprints are sent. This keeps uploads fast and avoids storing full playlists publicly. If you choose to submit a provider, that submission may be stored and displayed.

Why this helps

Many resellers copy the same channel list and sell it under different names. By comparing fingerprints and group lists we can detect when different providers are actually the same package — helping you find better prices or spot duplicated offers.

Tips
  • Use a provider website URL that starts with https:// or http://.
  • If parsing appears slow, please be patient — large M3Us can take a minute.
  • For privacy, do not include credentials in the provider URL.
How scores and matches are calculated

This explains, in plain language, how the site decides whether two IPTV packages are the same or similar.

1) File fingerprint (exact match)

We compute a short fingerprint (a deterministic hash) from the provider's playlist. If two files have the same fingerprint, they are considered an exact match — identical channel lists.

2) Channel and group comparison

For non-identical files we compare two things:

  • Channel overlap: how many individual channel URLs appear in both files.
  • Group overlap: how many channel groups (categories) are the same between files.
3) How the similarity score is built

We turn the overlaps into percentages and combine them into a single similarity score. A simple, easy-to-understand example we use:

  • Channel overlap contributes about 60% of the score.
  • Group overlap contributes about 40% of the score.

So if two files share 80% of channels and 50% of groups, the score would be roughly 0.6*80 + 0.4*50 = 68%.

4) Thresholds and match labels

We use the score to label results so they're easy to act on:

  • Match (likely identical): score >= 90%
  • Partial match: score between 50% and 90% — similar package, may differ by a few channels or updated links
  • No match: score < 50% — different package
5) Practical notes
  • Small differences in channel URLs (tracking parameters, trailing slashes) are normalized before comparing to avoid false mismatches.
  • Exact hash matches are the strongest signal — if the hash is identical, treat packages as the same.
  • We prefer conservative thresholds to avoid false positives; a high-scoring partial match is still worth reviewing manually.
Example: Two providers share 95% of channels but list groups differently; final score might be ~80–90% and show as a strong partial match — worth checking the provider pages and prices.
About large M3U files

Large M3U files can take some time to parse. If the page appears frozen after you choose a file, please be patient — the site is processing the playlist.

Xtream / Panel → M3U URL Generator

Use this to build a typical Xtream-style M3U URL when you only have panel credentials. This generator creates a common URL pattern; check your provider or panel for exact path/parameters.

If left empty the generator will add type=m3u_plus&include=all_groups to request an M3U with all groups listed.
Note: Download may take a while to start/initiate for large playlists with many groups and channels.
Click Download to attempt fetching the M3U and prompt a download; if the fetch is blocked, the link will open in a new tab for manual download.

How to use: Click Generate to build the Xtream M3U link, then click Download to fetch and save the .m3u file automatically. If the browser blocks the request or the download fails (CORS/network), the link will open in a new tab so you can save the playlist manually. Keep credentials private and don’t share them publicly.
